Definition of Ready (DoR)


Requisitos para que una historia pueda entrar a un sprint:

Historia escrita con formato Scrum: Como [rol], quiero [acción], para [valor].

Criterios de aceptación claros y medibles (checklist).

Historia estimada en story points (size: 1, 3, 5…).

Diseño/prototipo UI disponible si aplica.

Dependencias identificadas y resueltas o documentadas.




Definition of Done (DoD)


Requisitos técnicos para considerar una historia cerrada:

Implementación completada según criterios de aceptación.

Código revisado (Pull Request aprobado por al menos un compañero).

Cobertura mínima de tests unitarios: 80%.

Funcionalidad desplegada en entorno de staging para validación.

Documentación técnica o comentario actualizado en GitHub.

Issue cerrado y vinculado en el Project Board como Done.
